## 2. Topaz Video Upscale — Node 87

**Source:** screenshots от пользователя 2026-05-31 + recon submit.

**Цена:** 1410 кредитов за запуск (фиксировано в UI).

### V1.2 design decisions (2026-05-31)

По решению пользователя — упрощаем V1.2:

1. **`filter_enhancement_model` жёстко = `PROB4` (General / Proteus).**
   UI селектор семейств (Artemis / Nyx / Gaia / Iris / Themis / Dione) — V1.3+.
   В коде закреплено как `TopazUpscaleWorkflow.ENHANCEMENT_MODEL_FIXED`.
2. **Topaz НЕ участвует в `VideoScenario`** — у ноды 87 одна форма входа
   (`init_video` обязателен), сценарии тут излишни. Валидация — прямо в
   `topaz_upscale.py:build_payload` (одна строка `if not init_video: raise`).
3. **UI выставляет только 4 параметра** (+ interpolation toggle):
   - `output_upscale` (X1 / X2 / X3 / X4)
   - `output_crop_to_fit` (Auto / Yes / No → NULL / TRUE / FALSE)
   - `output_container` (mp4 / mov / mkv)
   - `filter_frame_interpolation_params` (off by default; раскрывает доп.
     селекторы model / slowmo / fps)
   - всё остальное — backend defaults.

### Inputs
```
init_video         type=video    value=<file_obj_id>   обязателен (это upscale, без видео нечего апскейлить)
```

### Params (все enum'ы — из `nodes_dump.json`)

| Параметр (backend) | Тип | Дефолт | Все значения |
|---|---|---|---|
| `output_upscale` | enum | `X2` | `X1, X2, X3, X4` |
| `filter_enhancement_params` | bool | `True` | UI: "Use Upscale" чекбокс |
| `filter_enhancement_model` | enum | `PROB4` | **24 кода — см. таблицу семейств ниже** |
| `filter_frame_interpolation_params` | bool | `False` | UI: "Use Frame Interpolation" чекбокс |
| `filter_frame_interpolation_model` | enum | `NULL` | `AION1, APF2, APO8, CHF3, CHR2, NULL` |
| `filter_frame_interpolation_slowmo` | number | `0` | UI: "Slow Motion Rate" |
| `filter_frame_interpolation_fps` | number | `0` | UI: "FPS" |
| `filter_frame_interpolation_duplicate` | enum | `NULL` | `TRUE, FALSE, NULL` (UI: Auto/Yes/No) |
| `filter_frame_interpolation_duplicate_threshold` | number | `0` | UI: "Duplicate Threshold" |
| `output_crop_to_fit` | enum | `NULL` | `TRUE, FALSE, NULL` → **UI Auto=NULL, Yes=TRUE, No=FALSE** |
| `output_container` | enum | `mp4` | `mp4, mov, mkv, NULL` |

### `filter_enhancement_model` — 24 кода, 7 UI-семейств

Семейства определены по префиксу кода. UI показывает только семейство —
backend-код, который UI шлёт при выборе семейства, выбираем по нашей логике
(обычно — самый новый/универсальный preset семейства). Точное соответствие
UI-семейство → конкретный backend-код проверим единичным GUI-кликом перед
релизом (по 1 submit на семейство).

| UI семейство | Кандидаты backend-кодов | Наш дефолт (предложение) |
|---|---|---|
| `---` (skip enhancement) | `NULL` | `NULL` |
| **General (Proteus)** | `PROB4` | `PROB4` ✓ (подтверждено recon'ом) |
| **Denoise and Sharpen (Artemis)** | `AAA9, AHQ12, ALQ13, ALQS2, AMQ13, AMQS2` | `AMQ13` (medium quality, recent) |
| **Denoise (Nyx)** | `NXF1, NYX3` | `NYX3` (новее) |
| **CG, GenAI, Animation (Gaia)** | `GCG5, GHQ5` | `GHQ5` (high quality) |
| **Faces (Iris)** | `IRIS2, IRIS3` | `IRIS3` (новее) |
| **Motion Deblur (Themis)** | `THD3, THF4, THM2` | `THM2` (motion-default по имени) |
| **Old Content (Dione)** | `DDV3, DTD4, DTDS2, DTV4, DTVS2` | `DTV4` (TV deinterlace, recent) |
| прочие | `RHEA1, RXL1` | назначение неизвестно — игнорируем в UI |

**Quality suffixes (паттерн внутри семейства Artemis):**
- `H` = High quality, `M` = Medium, `L` = Low
- `S` = Strong / Standard variant?
- цифра — версия модели

Перед релизом — один manual recon: переключить семейство в UI 7 раз →
проверить какой именно код UI шлёт. После этого `default_code` финализируется.

### Output
```
name="video", type="array"
```

Output array — Topaz, по всей видимости, может возвращать несколько вариантов
(enhancement + interpolated separately?). Уточнить при первом реальном run'е.
